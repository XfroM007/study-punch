## 二 · 关键 bug 修复记录

| # | Bug | 根因 | 修复 |
|---|---|---|---|
| 1 | `S.get is not a function` | 浏览器原生 `Storage` 构造函数覆盖了 `window.Storage` | shim.js 用 `Object.defineProperty` 强写 |
| 2 | `Unexpected token 'export'` | ES6 `export` 在 file:// + 非 module 加载下不识别 | sed 批量移除 export 关键字 |
| 3 | punch.js / ui.js 的 storage.js 引用未定义函数 | `var S = { updateToday, ... }` shorthand 在 var 声明时找不到后续函数 | shim.js 改用 `window.PunchStorage` 延迟引用 |
| 4 | agent-B / agent-C 接口不兼容 | 分派任务时未锁接口 | 决策：采用 agent-B 实现（已与 ui.js 配对），agent-C 接口作为独立 demo |

## 七 · 主 agent 自责（LRN 候选）

1. **分派任务时未锁接口** —— 让前端 agent 和数据 agent 各自实现，导致集成踩坑 1 小时
2. **subagent 用法失当** —— 大批量逐字写作不适合抛 subagent，agent 两次失败浪费 30 分钟
3. **汇报节奏失控** —— 中途有 30 分钟没主动同步进度，老刘等结果时焦虑（LRN-20260715-001）

**改进措施**：
- 多 agent 任务**必须先锁接口**（用 TypeScript 接口或 JSDoc 注释）
- 大批量内容写作（>3000 字）主 agent 亲自写，不抛 subagent
- subagent 派出后每 10 分钟主动汇报一次进度给老刘
