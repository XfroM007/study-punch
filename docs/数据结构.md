## 一 · 命名空间与版本策略

- **命名空间**：`study-punch:v1:` 前缀统一管理
- **版本演进**：v1 → v2 升级时，迁移脚本读取 v1 数据 + 写入 v2 + 标记 v1 `migrated=true`
- **写入策略**：所有写入都走 `JSON.stringify`，读取都走 `JSON.parse` + 异常兜底（损坏数据降级为初始值）

## 三 · 字段定义

### 3.1 `profile` — 用户资料

```js
{
  uuid: "sp_20260715_a1b2c3d4",   // 首次进入生成，格式：sp_<yyyymmdd>_<8位随机>
  createdAt: 1752537600000,         // 首次进入时间戳
  nickname: "学习者"                 // 可在设置里改
}
```

### 3.2 `progress` — 进度

```js
{
  streak: 0,            // 当前连续打卡天数
  maxStreak: 0,         // 历史最长
  totalDays: 0,         // 累计打卡天数
  score: 0,             // 累计积分
  ancientDone: [],      // 已完成古文 id 数组（去重）
  wordDone: [],         // 已掌握单词 id 数组（去重）
  lastPunchDate: null,  // YYYY-MM-DD，null 表示从未打卡
  wordCountTotal: 0     // 累计完成单词次数（用于单词成就：a08/100, a09/500, a10/1000）
}
```

### 3.3 `daily-log` — 每日打卡记录

```js
[
  {
    date: "2026-07-15",
    ancient: {                       // 古文记录（可为 null 表示跳过）
      id: "g01",
      duration: 180,                 // 秒
      errors: 0,                     // 错字数
      completed: true
    },
    words: [                         // 单词记录数组，长度 = 实际答题数
      { id: "e001", correct: true, attempts: 1 },
      { id: "e002", correct: false, attempts: 2 }
    ],
    score: 30,                       // 当日获得积分
    submittedAt: 1752541200000       // 完成时间戳
  }
]
```

**积分计算规则**（与项目定义 §三 保持一致 + 拓展）：
- 基础打卡：+10
- 古文完成（无错或 completed=true）：+10
- 单词全对（errors=0）：+10
- 当日满分（古文完成 + 单词全对 + 无错字）：+20（替代以上三项加成）
- 连击加成：streak ≥ 7 当日 +20（一次性结算）；streak ≥ 30 当日 +50（一次性结算）
- 实际公式：`score = 10 + (古文完成?10:0) + (单词全对?10:0) + (streak>=7?20:0) + (streak>=30?50:0)`
- **防刷**：同日多次 `completeDaily()` 仅首条生效（返回 `{already: true}`）

### 3.4 `wrong-words` — 生词本

```js
[
  {
    wordId: "e002",         // 单词 id（与 data/words.json 对齐）
    wrongCount: 2,          // 累计错次数
    lastWrong: 1752541200000 // 最近一次错的时间戳
  }
]
```

### 3.5 `achievements` — 已解锁成就

```js
["a01", "a02"]   // 只存 id 列表，定义在 ACHIEVEMENTS 常量里
```

### 3.6 `settings` — 设置

```js
{
  theme: "dark",         // "dark" | "light"（UI 主题，本版本默认 dark）
  dailyTarget: { ancient: 1, words: 20 },  // 每日目标（v1 锁死，仅作占位）
  reminder: {
    enabled: false,
    time: "20:00"        // HH:mm
  }
}
```

### 3.7 `backup` — 导出快照

导出时把上面 6 个 key（profile / progress / daily-log / wrong-words / achievements / settings）打包成一个对象，加 `_exportedAt` 与 `_schemaVersion`：

```js
{
  _schemaVersion: "v1",
  _exportedAt: 1752544800000,
  profile: { ... },
  progress: { ... },
  "daily-log": [ ... ],
  "wrong-words": [ ... ],
  achievements: [ ... ],
  settings: { ... }
}
```

### 3.8 `meta` — 内部元数据

```js
{
  schemaVersion: "v1",
  initializedAt: 1752537600000,
  migrated: null   // null | "from:v0" 记录迁移来源
}
```

## 四 · 打卡逻辑

### 4.1 今日任务分配（轮询策略）

避免每次都从 `g01` / `e001` 开始。算法：
1. `progress.ancientDone` 长度 = 已完成古文数 N
2. 今日古文 id = 今日 `daily-log` 中已包含的所有古文 id 之后第 1 个未完成；若无则回到第一篇
3. 单词：取所有未掌握（`!wordDone.includes(id)`）前 20 个；不足则补已掌握单词中错误率高的前 N 个
4. 边界：所有古文都学完 → 进入"复习模式"，按错误率排序取 1 篇
5. 单词同理

> 简单实现：基于"剩余未掌握优先 + 历史错误率加权"的轮询索引，由 `punch.js` 维护 `state.taskIndex`。

### 4.2 streak 规则（重点）

`completeDaily()` 伪代码：

```
INPUT: 无（默认今日 YYYY-MM-DD）
PROGRESS: progress.streak, progress.lastPunchDate, progress.maxStreak

IF lastPunchDate == today:
  RETURN { already: true }        // 同日重复，忽略
IF lastPunchDate == (today - 1 day):
  streak = streak + 1
ELSE:
  streak = 1                      // 跨天没打卡 / 首次打卡
lastPunchDate = today
maxStreak = max(maxStreak, streak)
totalDays = totalDays + 1
```

> **两阶段提交**：v1 把打卡拆成两步 ——
> 1. `completeDaily()`：只走 streak / totalDays / lastPunchDate / 占位 dailyLog，派发 `punch:daily-complete` 事件
> 2. `finishDaily(answers)`：前端传答案 `{ancient, words}`，写入积分、错词本、ancientDone/wordDone、wordCountTotal，再派发 `punch:daily-finish`
>
> 这样设计的好处是 streak 与「积分、错词累计」解耦，前端可以分步渲染（打卡按钮 + 答题卡 + 结算页）。

### 4.3 答题校验

- **古文**：`submitAncientAnswer(id, answer)` 与 `data/ancient.json` 中对应篇目逐字比对；返回 `{correct, score, errors[], totalChars}`。
  - v1 简化为：传完整字符串，校验是否完全一致；errors = 错字数 = Levenshtein 距离（或简化为按字比对计数）
- **单词**：`submitWordAnswer(id, answer)` 与 `data/words.json` 中 `word` 或 `translation` 比对（忽略大小写、去空格）；返回 `{correct, expected}`。

### 4.4 防刷与幂等

- `completeDaily()` 同一天多次调用：第二次起返回 `{already: true, score: 0}`
- `addWrongWord()` 累加而非新增
- `unlockAchievement(id)` 已解锁直接返回 `false`，未解锁则 push 并返回 `true`

## 五 · 成就系统（12 张卡牌）

定义在 `punch.js` 的 `ACHIEVEMENTS` 常量里，前端只读取 `id` 和 `unlocked` 字段。

| id | 名称 | 条件 | 图标建议 |
|---|---|---|---|
| a01 | 启程 | `totalDays >= 1` | 🚀 |
| a02 | 三日连斩 | `streak >= 3` | ⚔️ |
| a03 | 旬日连斩 | `streak >= 10` | 🔥 |
| a04 | 月度连斩 | `streak >= 30` | 👑 |
| a05 | 古韵初探 | `ancientDone.length >= 1` | 📜 |
| a06 | 古韵十篇 | `ancientDone.length >= 10` | 📚 |
| a07 | 古韵大成 | `ancientDone.length >= TOTAL_ANCIENT` | 🏛️ |
| a08 | 单词新芽 | `wordCountTotal >= 100` | 🌱 |
| a09 | 单词森林 | `wordCountTotal >= 500` | 🌳 |
| a10 | 单词海洋 | `wordCountTotal >= 1000` | 🌊 |
| a11 | 满分日 | 最近一条 daily-log 的 errors=0 且 words 全 correct | ⭐ |
| a12 | 千分达人 | `score >= 1000` | 💎 |

`checkAchievements()` 在 `completeDaily()` 末尾调用，遍历 `ACHIEVEMENTS`，未解锁且条件满足则调 `unlockAchievement()`，返回新解锁的 id 列表（用于前端弹卡动画）。

## 六 · 接口约定（agent-B 接入）

> agent-B（前端）只需调以下公开方法，无需直接读 localStorage。

### `window.PunchStorage`（storage.js）

```js
PunchStorage.init()                        // 首次进入初始化（幂等）
PunchStorage.getProfile()
PunchStorage.setProfile(patch)             // patch 合并
PunchStorage.getProgress()
PunchStorage.updateProgress(patch)         // patch 合并
PunchStorage.addDailyLog(entry)            // 返回写入后的完整条目
PunchStorage.addWrongWord(wordId)          // 累加并返回最新一条
PunchStorage.unlockAchievement(id)         // 返回 boolean
PunchStorage.getAchievements()             // 返回 id 列表
PunchStorage.getSettings()
PunchStorage.setSettings(patch)
PunchStorage.exportAll()                   // 返回 JSON 对象
PunchStorage.importAll(json)               // 返回 boolean
PunchStorage.resetAll()                    // 回到初始状态
PunchStorage.isInitialized()
```

### `window.Punch`（punch.js，依赖 PunchStorage）

```js
Punch.getTodayTask()                       // {date, ancient, words, isReviewMode}
Punch.submitAncientAnswer(id, answer)      // {correct, score, errors, totalChars, expected}
Punch.submitWordAnswer(id, answer)         // {correct, expected, id}
Punch.completeDaily()                      // {already, score:0, streak, totalDays, newAchievements, dailyLog, task}
Punch.finishDaily(answers)                 // {score, streak, totalDays, newAchievements, dailyLog}  // 二阶段提交
Punch.checkAchievements()                  // [achievementId, ...]
Punch.getAchievementList()                 // [{id, name, desc, icon, unlocked, unlockedAt}]
Punch.getStreakInfo()                      // {streak, maxStreak, totalDays, score, lastPunchDate}
Punch.getWrongWords()                      // 生词本全量
Punch.getTodayLog()                        // 今日已打卡则返回 daily-log 条目，否则 null
```

### 全局事件（可选）

`completeDaily()` 成功后派发：

```js
window.dispatchEvent(new CustomEvent('punch:daily-complete', {
  detail: { already, score: 0, streak, newAchievements, task }
}));
```

`finishDaily(answers)` 成功后派发：

```js
window.dispatchEvent(new CustomEvent('punch:daily-finish', {
  detail: { score, streak, totalDays, newAchievements, dailyLog }
}));
```

`checkAchievements()` 解锁新成就时派发：

```js
window.dispatchEvent(new CustomEvent('punch:achievement-unlock', {
  detail: { ids: ['a01', 'a05', ...] }
}));
```

agent-B 可监听这些事件触发粒子扩散 + 卡牌翻转动画。

## 七 · 错误兜底

- `JSON.parse` 抛错：视为空数据，重新初始化
- 关键字段缺失（如 `progress` 没 `streak`）：用 `DEFAULT_PROGRESS` 补齐
- `getTodayTask()` 在 `data/ancient.json` / `data/words.json` 缺失时返回空任务并 `console.warn`
- `completeDaily()` 失败：不写入 daily-log，保留 progress 原状

## 八 · 测试场景（验收）

1. **首次进入**：`init()` 后所有 key 都存在
2. **打卡 1 次 + 刷新**：progress.streak=1, totalDays=1, score=30, lastPunchDate=今日
3. **同日重复打卡**：返回 `{already: true}`，score 不变
4. **跨天测试**（手改 `lastPunchDate` 为昨日）：再打卡 streak=2
5. **跨多天**（手改 lastPunchDate 为 5 天前）：再打卡 streak=1（重置）
6. **导出 → reset → 导入**：所有数据完全恢复
7. **成就触发**：
   - 首次打卡 → a01 解锁
   - streak=3 → a02
   - ancientDone.length=10 → a06
   - 单词全对 + 古文无错 → a11
